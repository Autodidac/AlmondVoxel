AlmondVoxel quick start

Cross-platform build notes for Visual Studio users

Prerequisites

  • Visual Studio 2022 with the Desktop development with C++ workload.
  • Git for Windows (provides Git Bash for running the helper scripts).
  • CMake 3.23+ and Ninja (install via the Visual Studio installer or package managers such as Chocolatey).
  • SDL3 development package from vcpkg (vcpkg install sdl3) for the terrain_demo.

Configure and build

  1. Open Developer PowerShell for VS or x64 Native Tools Command Prompt.
  2. Clone the repository: git clone https://example.com/AlmondVoxel.git.
  3. Configure the build tree:
    bash ./cmake/configure.sh msvc Debug
  4. Compile the demos, tests, and benchmarks:
    bash ./build.sh msvc Debug

Run demos and tests

Use run.sh to start a target from any shell:

bash ./run.sh terrain_demo
bash ./run.sh almond_voxel_tests
bash ./run.sh mesh_bench

Each command searches the generated build folders and launches the first matching executable.

Headless or remote builds

When working on a headless agent, disable the SDL3 demo during configuration:

bash ./cmake/configure.sh msvc Release -DALMOND_VOXEL_BUILD_EXAMPLES=OFF

The examples can be re-enabled locally by reconfiguring with the default options.

Additional notes

  • The project is maintained privately; external contributions and pull requests are not accepted.
  • Refer to README.md for a module overview and cross-platform build matrix.
  • Check docs/windows.md for troubleshooting guidance specific to MSVC environments.
  • Linux or macOS builds follow the same scripts with the gcc or clang arguments.
